# BiteFit Pro — תוכנית מנוי (Plan)

> מסמך תכנון. נכון ליוני 2026. אין עדיין חשבונות מפתחים — זה ה-blueprint לפני בנייה.

---

## 1. מה Pro כולל

| פיצ'ר | חינם (Free) | Pro |
|--------|:-----------:|:---:|
| תיעוד מזון ידני | ✅ ללא הגבלה | ✅ |
| מעקב משקל + מים | ✅ | ✅ |
| יעד קלוריות/מאקרו יומי בסיסי | ✅ | ✅ |
| **צילום מזון AI** | **3 / יום** | ♾️ ללא הגבלה |
| **צ'אט Biti** | **30 / יום** | ♾️ ללא הגבלה |
| **מנוע ההתאמה** (תוכנית מסתגלת, יעד לכל ארוחה, TDEE מסתגל) | ❌ | ✅ |
| **הצעות מתכונים מותאמות** | ❌ | ✅ |
| היסטוריה וניתוחים | 7 ימים אחרונים | מלא |

**העיקרון:** החינם נותן טעימה אמיתית (תיעוד מלא + מעט AI); Pro פותח את ה-AI הבלתי-מוגבל ואת ההתאמה האישית — שהם גם הערך הגבוה וגם העלות האמיתית.

## 2. תמחור

| תוכנית | מחיר | הערה |
|--------|------|------|
| חודשי | **₪29 / חודש** | גמישות |
| שנתי | **₪199 / שנה** | ~₪16.6/חודש — חיסכון 43%, מזומן מראש, שימור טוב יותר |
| ניסיון | **7 ימים חינם** | על שתי התוכניות — מעלה המרה |

עמלת אפל/גוגל: 15% (Small Business, מתחת ל-$1M/שנה) → נטו ~₪25/חודש.
עלות AI למשתמש Pro פעיל: ~₪2/חודש → **רווח גולמי ~90%**.

---

## 3. איך התשלום עובד (השרשרת)

```
משתמש לוחץ "שדרג" → חלון תשלום מובנה של Apple/Google →
Apple/Google גובים + לוקחים עמלה → RevenueCat מקבל אישור ומסמן entitlement="pro" →
השרת מעדכן profile.tier="pro" → הגבלת הקצב נותנת ל-Pro הכל ללא הגבלה
```

- אסור Stripe/אשראי ישיר באפליקציה — חובה IAP של אפל/גוגל.
- IAP לא עובד ב-Expo Go — דרוש EAS Build (גרסה אמיתית).

## 5. מה כבר מוכן בצד שלנו ✅
- מדידת שימוש + הגבלת קצב (`api/usage.py`) — 3 צילומים/יום, 30 צ'אט/יום בחינם
- השרת בודק `tier` ופותח הכל ל-Pro
- טבלת `usage_daily` ב-Supabase (RLS)
- דחיסת תמונה + cache להערכת תזונה (חוסכים עלות)

## 6. מה חסר (לבנייה עתידית)
- חשבונות מפתחים (צעדים 1-2) ← תלוי בך
- EAS Build + RevenueCat SDK (צעד 5)
- מסך Paywall (צעד 6)
- Webhook entitlement→tier (צעד 7)
- שדה `tier` נשמר ב-`meal_preferences` JSONB — צריך endpoint/webhook שמעדכן אותו לאחר רכישה

---

## 7. הצעד הבא המומלץ
לפתוח את חשבון **Apple Developer** (הכי איטי — אימות זהות לוקח ימים). במקביל אפשר לבנות את **מסך ה-Paywall** (ויזואלי, בלי תשלום אמיתי) כדי לאמת UX. החיבור האמיתי מחכה לחשבונות + EAS Build.
